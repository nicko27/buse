Documentation UpdateFlow

Aperçu

UpdateFlow est un système moderne de gestion des mises à jour basé sur Git qui fournit une interface robuste pour gérer les opérations de contrôle de version en PHP (backend) et JavaScript (frontend).

Architecture

1. Backend (PHP)

L'implémentation PHP est organisée en plusieurs composants :

Classe Principale (UpdateFlow.php)

  • Gère les opérations Git et le contrôle de version
  • Gère la validation de la configuration
  • Fournit les interfaces principales pour les opérations pull/push
  • Inclut la gestion des erreurs et la journalisation

Composants

  • Handlers/
    • PullHandler : Gère les opérations de pull
    • PushHandler : Gère les opérations de push
  • Managers/
    • GitManager : Gère les opérations Git
    • VersionManager : Gère les numéros de version
  • Utils/
    • Logger : Gère la journalisation
    • Response : Standardise les réponses API

2. Frontend (JavaScript)

L'implémentation JavaScript fournit une interface côté client avec :

  • Communication API asynchrone
  • Mécanisme de réessai pour les requêtes échouées
  • Protection CSRF
  • Gestion des événements
  • Journalisation de débogage
  • Suivi des versions

Configuration

Configuration Backend

{
    'repoPath': string,      // Requis: Chemin vers le dépôt Git
    'versionPath': string,   // Requis: Chemin vers le fichier de version
    'logger': array,         // Requis: Configuration du logger
    'versionFormat': string, // Optionnel: Format du fichier de version (défaut: 'json')
    'maxRetries': int,       // Optionnel: Nombre maximum de tentatives (défaut: 3)
    'retryDelay': int,       // Optionnel: Délai entre les tentatives en ms (défaut: 1000)
    'timeout': int,          // Optionnel: Timeout des requêtes en ms (défaut: 30000)
    'debug': bool,           // Optionnel: Mode debug (défaut: false)
    'allowedBranches': array // Optionnel: Liste des branches autorisées
}

Configuration Frontend

{
    apiEndpoints: {           // Requis: URLs des endpoints API
        pull: string,
        push: string
    },
    versionEndpoint: string,  // Requis: Endpoint de vérification de version
    autoReload: boolean,      // Optionnel: Rechargement auto après mise à jour
    retryAttempts: number,    // Optionnel: Nombre de tentatives
    retryDelay: number,       // Optionnel: Délai entre les tentatives
    timeout: number,          // Optionnel: Timeout des requêtes
    debug: boolean,           // Optionnel: Activation du debug
    csrfToken: string        // Optionnel: Token CSRF
}

Utilisation

Implémentation PHP

// Initialisation UpdateFlow
$updateFlow = new UpdateFlow([
    'repoPath' => '/chemin/vers/repo',
    'versionPath' => '/chemin/vers/version.json',
    'logger' => ['path' => '/chemin/vers/logs']
]);

// Obtenir la version actuelle
$version = $updateFlow->getVersion();

// Tirer les mises à jour
$pullResult = $updateFlow->pull();

// Pousser les changements
$pushResult = $updateFlow->push('message de commit', 'patch|minor|major');

Implémentation JavaScript

// Initialisation UpdateFlow
const updateFlow = new UpdateFlow({
    apiEndpoints: {
        pull: '/api/update/pull',
        push: '/api/update/push'
    },
    versionEndpoint: '/api/version'
}, {
    success: (msg) => console.log(msg),
    error: (msg) => console.error(msg)
});

// Tirer les mises à jour
await updateFlow.pull();

// Pousser les changements
await updateFlow.push('message de commit', 'patch');

Sécurité

  • Protection CSRF
  • Validation des requêtes
  • Restrictions des branches
  • Support d'authentification
  • Assainissement des entrées

Gestion des Erreurs

  • Classes d'exception personnalisées
  • Mécanisme de réessai pour les requêtes échouées
  • Journalisation détaillée des erreurs
  • Réponses d'erreur standardisées

Bonnes Pratiques

  1. Toujours fournir des messages de commit explicites
  2. Utiliser les types de version appropriés (patch/minor/major)
  3. Activer le mode debug pendant le développement
  4. Configurer une journalisation appropriée
  5. Définir des valeurs de timeout appropriées
  6. Implémenter une gestion d'erreurs adéquate

Gestion des Versions

Suit le versionnement sémantique (MAJOR.MINOR.PATCH) :

  • PATCH : Corrections de bugs et petits changements
  • MINOR : Nouvelles fonctionnalités (rétrocompatibles)
  • MAJOR : Changements incompatibles
Note : Assurez-vous de suivre strictement ces conventions de versionnement pour maintenir la cohérence du projet.
