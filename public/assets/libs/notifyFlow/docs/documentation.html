Documentation NotifyFlow

Introduction

NotifyFlow est une bibliothèque JavaScript moderne et flexible pour afficher des notifications non bloquantes (toasts) dans vos applications web. Elle offre une personnalisation facile de l'apparence, de la position, de la durée, des animations et permet l'ajout d'actions interactives.

Caractéristiques principales :

  • Types de notifications prédéfinis (info, succès, avertissement, erreur).
  • Positionnement configurable (coins, centre haut/bas).
  • Durée d'affichage personnalisable ou infinie.
  • Animations d'entrée/sortie (fade, slide).
  • Possibilité d'ajouter des boutons d'action avec callbacks.
  • Exécution automatique d'une action après un délai.
  • Barre de progression visuelle pour la durée.
  • Thèmes clair/sombre ou adaptation automatique aux préférences système.
  • Gestion de l'empilement et nombre maximum de notifications.
  • Style visuel "glassmorphism" moderne.

Installation

Pour utiliser NotifyFlow, incluez son fichier CSS et son fichier JavaScript dans votre page HTML.

<!-- Feuille de style de NotifyFlow -->
<link rel="stylesheet" href="path/to/notifyFlow.css">

<!-- Script de NotifyFlow -->
<script src="path/to/notifyFlow.js"></script>
Le script notifyFlow.js expose la classe NotifyFlow globalement (window.NotifyFlow) ou peut être utilisé avec des systèmes de modules (CommonJS, AMD).

Initialisation

Créez une instance de NotifyFlow pour commencer à afficher des notifications. Vous pouvez passer un objet de configuration au constructeur pour définir des options globales par défaut.

// Initialisation simple (utilise les options par défaut)
const notify = new NotifyFlow();

// Initialisation avec configuration personnalisée
const customNotify = new NotifyFlow({
    position: 'bottom-left', // Position par défaut
    theme: 'dark',          // Thème sombre par défaut
    defaultDuration: 5000,  // Durée par défaut de 5s
    maxNotifications: 3,    // Afficher 3 notifications max simultanément
    animations: {
        in: 'slide-in',     // Animation d'entrée
        out: 'fade-out'     // Animation de sortie
    },
    debug: false            // Désactiver les logs de debug
});

OptionTypeDéfautDescription

themestring'light'Thème visuel. Valeurs possibles : 'light', 'dark', 'auto' (s'adapte aux préférences système).

Afficher une Notification

Méthode Principale : notify(options)

La méthode notify() est la plus flexible pour créer et afficher une notification. Elle prend un objet d'options en argument.

notify.notify({
    title: 'Mise à jour disponible', // Titre optionnel
    message: 'Une nouvelle version est prête à être installée.', // Message principal (requis)
    type: 'info', // Type de notification (info, success, warning, error)
    icon: '<i class="fas fa-cloud-download-alt"></i>', // Icône HTML/SVG optionnelle
    duration: 10000, // Durée spécifique (10s)
    progress: true, // Afficher la barre de progression
    dismissible: true, // Afficher le bouton de fermeture
    actions: [ // Boutons d'action optionnels
        {
            id: 'install',
            text: 'Installer',
            class: 'primary', // Classe CSS pour le bouton
            callback: () => { console.log('Installation lancée...'); return true; }, // Retourne true pour fermer
            dismiss: false // Optionnel : false pour ne pas fermer au clic
        },
        { id: 'later', text: 'Plus tard', callback: () => console.log('Reporté.') }
    ],
    autoAction: 'later', // ID de l'action à exécuter après le délai
    autoActionDelay: 8000, // Délai pour l'autoAction (8s)
    customClass: 'update-notification' // Classe CSS personnalisée
});

Méthodes Raccourcis

Pour plus de commodité, des méthodes raccourcis existent pour chaque type de notification standard. Elles prennent le message, un titre optionnel, et un objet d'options (qui surcharge les options par défaut et celles du constructeur).

// Notification de succès
notify.success('Opération réussie !', 'Succès');

// Notification d'information avec durée personnalisée
notify.info('Ceci est une information importante.', 'Info', { duration: 10000 });

// Notification d'avertissement sans fermeture automatique
notify.warning('Attention, action irréversible.', 'Avertissement', { duration: 0 });

// Notification d'erreur avec une classe CSS perso
notify.error('Impossible de contacter le serveur.', 'Erreur Réseau', { customClass: 'network-error' });

// Notification spéciale 'warningNotice' (style et icône prédéfinis)
notify.warningNotice('Votre session expirera bientôt.', 'Session');

Actions Personnalisées

Vous pouvez ajouter des boutons interactifs à vos notifications via l'option actions. C'est un tableau d'objets, où chaque objet représente un bouton.

Structure d'un objet Action

PropriétéTypeDescription
idstringRequis. Identifiant unique de l'action (utilisé pour `autoAction`).
textstringRequis. Texte affiché sur le bouton.
callbackfunctionRequis. Fonction exécutée lors du clic. Si elle retourne explicitement false, la notification ne se fermera pas automatiquement.
classstring(Optionnel) Classe(s) CSS additionnelle(s) pour le bouton.
dismissboolean(Optionnel) Si false, empêche la fermeture automatique de la notification après l'exécution du callback. Défaut : true.

Action Automatique

Utilisez autoAction (avec l'ID de l'action) et autoActionDelay (en ms) pour déclencher une action automatiquement après un certain temps. Un compte à rebours s'affiche sur le bouton concerné.

notify.notify({
    message: 'Redirection dans 5 secondes...',
    type: 'info',
    duration: 0, // Reste affichée
    actions: [
        {
            id: 'redirectNow',
            text: 'Aller maintenant',
            callback: () => window.location.href = '/nouvelle-page'
        },
        {
            id: 'cancelRedirect',
            text: 'Annuler',
            callback: () => { /* Annuler la redirection (logique applicative) */ }
        }
    ],
    autoAction: 'redirectNow', // ID de l'action à lancer
    autoActionDelay: 5000      // Après 5 secondes
});

Thèmes

NotifyFlow supporte trois modes de thème, configurables globalement via l'option theme du constructeur :

  • light (défaut) : Thème clair.
  • dark : Thème sombre.
  • auto : S'adapte automatiquement aux préférences de l'utilisateur (via la media query CSS prefers-color-scheme).

Le style général utilise un effet "glassmorphism" avec des arrière-plans semi-transparents et floutés.

Animations

Les animations d'entrée et de sortie sont configurables globalement via l'option animations du constructeur.

new NotifyFlow({
    animations: {
        in: 'slide-in', // 'slide-in' ou 'fade-in'
        out: 'fade-out' // 'fade-out' ou 'slide-out'
    }
});

Vous pouvez créer des animations personnalisées en CSS en ciblant les classes correspondantes (ex: .notify-flow-item.custom-in-animation) et en utilisant les noms d'animation dans la configuration.

Personnalisation CSS

Personnalisez l'apparence en ciblant ces classes CSS principales :

  • .notify-flow: Le conteneur principal (positionné en fixed).
  • .notify-flow.top-right, .top-left, etc. : Classes de positionnement.
  • .notify-flow.light, .dark: Classes de thème.
  • .notify-flow-item: Chaque notification individuelle.
  • .notify-flow-item.info, .success, .warning, .error, .warningNotice: Classes de type.
  • .notify-flow-icon: Conteneur de l'icône.
  • .notify-flow-content: Conteneur du titre et du message.
  • .notify-flow-title: Le titre.
  • .notify-flow-message: Le message.
  • .notify-flow-close: Le bouton de fermeture (X).
  • .notify-flow-progress: La barre de progression.
  • .notify-flow-actions: Conteneur des boutons d'action.
  • .notify-flow-btn: Chaque bouton d'action.
  • .notify-flow-btn.primary, .secondary, etc. : Classes personnalisées pour les boutons.
  • .auto-action-timer: Le compteur sur un bouton autoAction.
  • Animations : .slide-in, .fade-out, etc.
